# 项目剩余问题修复设计

## 目标

修复本轮复评确认的全部本地问题：北交所 `920xxx` 股票代码被错误路由到沪市、五个独立工具缺少最低可靠性测试、GitHub Actions 只在运行器默认 Python 上执行检查。

本轮不发布、不 push、不创建 PR，也不改写研究报告或用户现有未提交内容。

## 修改范围

### 1. 北交所 920 代码路由

以 `tools/ashare_data.py` 的市场标准化结果作为唯一判断依据，并让以下调用统一使用该结果：

- 东方财富 `SECUCODE`：`920xxx.BJ`。
- 腾讯行情代码：`bj920xxx`。
- 东方财富行情 `secid`：`0.920xxx`。
- 东方财富财务数据过滤条件：`SECUCODE="920xxx.BJ"`。
- 东方财富搜索结果：当前接口返回 `MktNum="0"` 时显示为“北”。

显式市场后缀仍沿用现有行为；本轮只修复无后缀 920 代码的错误推断，不扩展到证券代码体系重构。

### 2. 五个工具的离线行为测试

使用 Python 标准库 `unittest`，不访问网络、不新增依赖。测试覆盖实际业务逻辑，而非仅验证模块可导入：

- `momentum_backtest.py`：动量触发与不触发、最近财报选择、价值评分。
- `momentum_backtest_v2.py`：JSON 行情解析、动量触发、财报前后关系和评分。
- `morningstar_fair_value.py`：Ticker 解析、有效记录筛选与涨幅排序、空数据结果；若空数据会导致除零，则一并修复。
- `star_history_chart.py`：单点/双点/多点曲线路径、线性与时间刻度、数字标签。只新增测试，不修改用户保留的执行权限变更和文件内容。
- `stock_screener.py`：动量判断、无基本面观察状态和各档信号分级。

只在测试首先证明存在真实错误时修改生产实现；不为提高测试数量重构脚本。

### 3. CI Python 版本矩阵

更新 `.github/workflows/check.yml`：

- 使用官方 `actions/setup-python@v6`，显式设置 Python，避免依赖运行器默认版本。
- 测试 Python 3.8（`ashare_data.py` 声明的最低版本）、3.11（常用中间版本）和 3.14（2026-07-16 的最新稳定特性版本）。
- 每个矩阵项运行现有 `bash scripts/check.sh`，不新增 coverage 或安装步骤。
- 设置 `fail-fast: false`，让所有版本的结果都可见。

## 测试先行顺序

1. 先为 920 的四条错误路由和搜索标签编写失败测试。
2. 实施最小路由修复，使新增测试通过。
3. 为五个工具逐一添加离线行为测试；若测试暴露真实边界错误，先保留失败证据，再做最小修复。
4. 更新 CI 矩阵。
5. 运行单文件测试、完整单元测试、`bash scripts/check.sh` 和 `git diff --check`。
