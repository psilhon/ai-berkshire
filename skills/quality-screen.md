# 去劣筛选：7条指标快速排除非一流公司

对 $ARGUMENTS 执行去劣指标筛选，快速排除不符合一流公司标准的标的。

**支持输入格式**：

| 输入方式 | 示例 | 说明 |
|---------|------|------|
| 个股 | `腾讯, 美团, 英伟达` | 逐家筛选 |
| 行业 | `中国啤酒行业` `全球云计算` `港股运动品牌` | 先搜索该行业主要上市公司（10-20家），再逐家筛选 |
| 市场/指数 | `恒生指数成分股` `沪深300` `纳斯达克100` | 拉取成分股列表，逐家筛选 |
| 主题 | `中国高股息50强` `全球AI算力链` | 先搜索主题相关公司，再逐家筛选 |

行业/市场/主题模式下，输出额外包含：通过率统计、行业内排名、板块对比总结。

## 设计原则

- **目标**：不错杀任何一流好公司，但能排除确定的非一流公司
- **逻辑**：7条硬指标 + 2条豁免规则，宁可漏网不可误杀
- **适用范围**：所有上市公司（金融股的指标适用性调整见下方「金融股口径」节）

## 7条去劣指标

| # | 指标 | 排除条件 | 衡量的是什么 |
|---|------|---------|-------------|
| 1 | 10年平均ROE | < 8% | 资本效率——股东的钱能不能跑赢机会成本 |
| 2 | 5年累计自由现金流 | 为负 | 真金白银——利润是不是"纸面富贵" |
| 3 | 利息覆盖倍数（EBIT/利息） | < 2倍 | 偿债安全——还利息的能力 |
| 4 | 长期毛利率 | < 15% | 定价权——产品/服务有没有差异化 |
| 5 | 经营现金流 / 净利润（5年均值） | < 0.7 | 利润质量——赚到的利润能不能收回现金 |
| 6 | 长期净利率 | < 5% | 抗风险能力——收入波动时利润是否归零 |
| 7 | 5年总股本膨胀 | > 20%（非并购原因） | 股东利益——管理层是否在稀释你的权益 |

## 3条豁免规则

### 豁免A：战略投入期豁免（适用于第1条）

如果同时满足以下3个条件，可豁免第1条ROE不达标：
1. 上市不足10年
2. 毛利率 > 30%（证明商业模式本身有定价权）
3. 最近2年经营现金流为正（证明造血能力已具备）

**逻辑**：高毛利率+现金流转正说明商业模式没问题，ROE低只是因为还在投入期。典型案例：美团。

### 豁免B：主动低利润率豁免（适用于第6条）

如果同时满足以下2个条件，可豁免第6条净利率不达标：
1. 毛利率 > 30%（有能力赚但选择不赚）
2. 近2年净利率已回升至5%以上，或呈明确上升趋势

**逻辑**：毛利率高说明有定价权，净利率低是战略选择（再投资）而非能力缺失。典型案例：亚马逊。

### 豁免C：高周转薄利模式豁免（适用于第4条和第6条）

如果同时满足以下3个条件，可豁免第4条毛利率和第6条净利率不达标：
1. ROE > 20%（证明虽然利润率低，但资本回报率极高）
2. 经营现金流/净利润 > 1.0（利润质量无问题）
3. 商业模式属于"会员制/平台佣金/高周转薄利"类型（利润不体现在商品加价上）

**逻辑**：有些一流公司的利润不藏在毛利率里，而是藏在会员费、周转效率或平台抽成中。它们的毛利率和净利率天然很低，但ROE极高说明资本效率一流。典型案例：Costco（毛利率12%、净利率2.5%，但ROE 25%+、会员续费率90%+）。

## 金融股口径（银行/保险/券商）

7条指标是为工业/消费企业设计的，金融股按下表调整适用性，**判决主要依托 ①⑥⑦ 三条**：

| 指标 | 银行/保险 | 券商 | 处理方式 |
|------|----------|------|---------|
| ③利息覆盖 | 不适用 | 不适用（延伸解释） | 利差/资金经营是商业模式本身；券商的两融、自营同为利差资金经营，同理豁免——这是对原规则的延伸解释，输出时须标注 |
| ④毛利率 | 不适用 | 不适用 | 金融股财报无毛利率科目，标"—"，不作通过/排除 |
| ②FCF ⑤OCF/NI | 结构性失真 | 结构性失真 | 银行 OCF 由存贷款变动主导、券商由客户保证金和自营盘变动主导，与利润质量无关（例证：招商银行 2016–2018 年 OCF 连续为负）。字面数值照列（标 ⚠️ 加脚注"仅列示不判决"），**不作为排除依据** |
| ①ROE ⑥净利率 ⑦稀释 | 正常适用 | 正常适用 | 金融股判决的主战场 |

**输出要求**：报告开头须先放一段"金融股口径说明"（先说清楚再打分），汇总表不适用项记"—"、失真项记"⚠️ 数值\*"加脚注。

## 执行流程

### 第一步：解析输入，确定筛选范围

**模式判断**：
- 如果输入是具体公司名/代码 → **个股模式**，直接进入第二步
- 如果输入是行业/市场/主题 → **批量模式**，先执行以下操作：
  1. 用 WebSearch 搜索该行业/市场/主题下的主要上市公司
  2. 行业模式：覆盖该行业市值前15-20家上市公司
  3. 指数模式：拉取完整成分股列表
  4. 主题模式：搜索相关公司，覆盖15-30家
  5. 列出完整公司清单供确认（如公司数>30，分批并行处理）

对每家公司确定全称、代码、交易所。

🔴 STOP / 检查点：在启动后台 Agent 批量取数或调用 A股数据管线/WebSearch 自主取数前，必须先向用户确认（给出明确选项：如"确认启动批量取数""仅先列出筛选公司清单不取数"），获得明确同意后再继续；未经确认不得自主执行。

### 第二步：并行数据收集

需要收集的数据（每家公司）：

1. **ROE**：近10年（或上市以来）的逐年ROE，计算平均值
2. **自由现金流**：近5年的经营现金流和资本开支，计算5年累计FCF
3. **利息覆盖**：最新年度EBIT和利息支出，计算倍数
4. **毛利率**：近5年毛利率趋势
5. **经营现金流/净利润**：近5年的比值，计算均值
6. **净利率**：近10年净利率趋势，计算均值
7. **总股本变化**：5年前和当前的总股本，计算膨胀比例

**A股标的优先走本仓库数据管线**（比 WebSearch 准且快，一次拿齐 10 年+）：

```bash
# 行情/市值/近5年核心财务
python3 tools/ashare_data.py quote <代码>
python3 tools/ashare_data.py financials <代码>
python3 tools/ashare_data.py valuation <代码>

# 10年年报指标：ROE、毛利率、净利率、OCF/净利润、利息覆盖、OCF
python3 tools/ashare_data.py history <代码> --years 10

# 第7条股本变化：独立股本变动史（含变动原因：增发/回购/配售）
python3 tools/ashare_data.py equity-history <代码>
```

⚠️ **陷阱**：`RPT_F10_FINANCE_MAINFINADATA` 返回的 `TOTAL_SHARE` 是**当前股本盖到所有历史行的静态值**，禁止用于第7条；历史股本必须走 `RPT_F10_EH_EQUITY` 股本变动史，或用历年"净利润÷EPS"反推交叉验证。

均值/偏差/膨胀率计算一律用 `python3 tools/financial_rigor.py calc --expr '...'`（精确 Decimal）。

WebSearch 的角色：非 A 股标的的数据收集；A 股关键数据（尤其决定通过/排除的那条指标）的**第二来源交叉验证**（按 financial-data 规范：东财主源 + 年报/巨潮副源，误差 ≤1% 才算一致）。

#### 批量模式并行执行策略

行业/指数/主题模式下，按以下规则并行取数：

| 参数 | 规则 | 值 |
|------|------|-----|
| **最大并发 Agent** | 同时运行的取数 Agent 上限 | 8 |
| **单 Agent 超时** | 每家公司的取数时限 | 120 秒 |
| **分批次阈值** | 超过此数分批发 | >30 家公司 |
| **每批间隔** | 批次间冷却时间 | 10 秒 |

**执行步骤**：
1. 将公司清单拆分为 ≤8 的批次（若 >30 家，每批 8 家分多批）
2. 每批内 8 个 Agent **并行启动**（同一条消息同时 spawn）
3. 等待该批全部完成后，检查结果：
   - 超时/失败的 Agent → 记录为"数据不足"，该标的标记 ⚠️
   - 成功的 Agent → 提取数据，进入下一步验算
4. 确认该批完成后，启动下一批（如有）
5. 全部批次完成后，用 `tools/financial_rigor.py calc` 统一验算各指标均值

**单 Agent 任务内容**（每个 Agent 对一家公司执行）：
1. 判断交易所：A 股 → 走 `ashare_data.py` 管线；其他 → WebSearch
2. A 股管线取数：`quote` + `financials` + `valuation` + `history --years 10` + `equity-history`（5 个子命令依次执行）
3. 非 A 股：WebSearch 搜索该公司的 ROE/FCF/毛利率/净利率/利息覆盖/OCF 趋势
4. 返回结构化结果：公司名 → 7 条指标逐条数据摘要

> 如果批量模式下可启动的 Agent 总数受运行时限制，降级为「分 3 批，每批按市值排序优先前 8 家」；未取到的标的标为 ⚠️ 并在报告中单独说明覆盖不全。

数据来源优先级：公司年报 > 券商研报 > 财经数据平台

### 第三步：逐条检验

对每家公司，逐条检验7个指标：
- ✅ 通过
- ❌ 未通过
- ⚠️ 边界（附数值说明）

如果触犯某条，检查是否满足对应豁免条件。

🔴 STOP / 检查点：在输出"通过/排除/豁免通过"等筛选强结论与板块质量判断前，必须先向用户确认（给出明确选项：如"输出完整筛选结果与通过率""仅输出中性分析不替用户下排除结论"），获得明确同意后再继续；未经确认不得自主替用户决定标的是否"非一流"。

## 注意事项

1. **金融股（银行/保险/券商）**：指标适用性按上方「金融股口径」节调整，判决主要依托①⑥⑦
2. **REIT**：ROE可能因物业重估波动大，用"核心营运利润ROE"代替
3. **数据不足**：如果某项数据无法获取，标注为"数据不足"而非直接判定通过/不通过
4. **周期性行业**：用完整周期（至少覆盖一个高点和一个低点）的平均值，不用单一年份。**均值离阈值 ±1 个百分点以内的边界过线/边界不过，必须在"边界争议"节做窗口敏感性披露**：换一个起止年（如 2015–2024 vs 2016–2025）结论是否翻转、是否靠单一牛市年份把均值拉过线。典型案例：广发证券 10 年 ROE 均值 8.62% 勉强过线，但靠的是 2025 牛市年，换窗口即翻转——这类"边界通过"必须与招商银行式"最差年份都远超阈值"的干净通过在措辞上明确区分（通过 ≠ 一流）
5. **上市时间短**：不足5年的公司用全部可得数据，但在结果中标注"数据窗口不足"

## 局限性声明

这套指标能排除"确定不好"的公司，但通过筛选不等于"确定好"。通过的公司仍需进一步研究：
- 商业模式是否可持续
- 管理层是否值得信赖
- 当前估值是否合理
- 竞争格局是否在恶化

去劣是第一步，不是最后一步。

## 反例与红线（不要做）

- 不要用单一信源判定通过/排除：A股关键指标（尤其决定结论的那条）必须双源交叉验证（东财主源+年报/巨潮副源，误差≤1%），单一来源不得作为排除依据。
- 不要用 `RPT_F10_FINANCE_MAINFINADATA` 的 `TOTAL_SHARE` 算第7条：它是当前股本盖到历史行的静态值，历史股本必须走 `RPT_F10_EH_EQUITY` 股本变动史或历年"净利润÷EPS"反推。
- 不要误杀一流公司：判决靠 7 条硬指标+3 条豁免，边界过线须做窗口敏感性披露，不得把"勉强过线"等同"一流"。
- 不要因数据不足直接判定排除：某指标无法获取应标"数据不足"，而非直接判不通过。
- 不要在用户未授权时落盘或外发：若本 skill 后续扩展为写入筛选报告，必须经过 D4 检查点获得明确同意（当前以对话输出为主，外发/落盘须确认）。
- 🔴 **不要用「~」近似值**：所有指标必须使用精确数值（保留到小数点后2位或原始精度），禁止写"~9.3%"、"约0.75x"等——必须从 `ashare_data.py` 的 stdout 或 Tushare API 返回值中取精确数字。
- 🔴 **不要产出薄摘要**：每个指标必须写出逐年精确数据、逐年判定、逐年理由；汇总表不能替代逐项分析。

## 失败处理（如果 X 失败 → Y）

- 如果 `tools/ashare_data.py` 取数失败/超时 → 执行 降级为 WebSearch 取数或提示用户检查代码/网络，并标注"数据来源：WebSearch，置信度中"，不静默跳过。
- 如果 `tools/financial_rigor.py calc` 报错（表达式/单位错误）→ 执行 检查表达式与小数精度后重算；仍失败则标注"计算未核验"且不在结论中使用该数值。
- 如果 双源交叉验证冲突（误差>1%）→ 执行 在结果中同时标注两源数字与差异，标为"证据不足"并要求补齐，不强行判定。
- 如果 WebSearch 取不到某行业/主题公司清单（批量模式）→ 执行 明确告知用户"无法获取完整清单"并列出已知部分，标注"覆盖不全"，不假装完整。
- 如果 用户输入含糊/多义（如仅给主题名）→ 执行 列出 2-3 个可能的行业/主题解读请用户澄清，不在未确认时自选展开。
